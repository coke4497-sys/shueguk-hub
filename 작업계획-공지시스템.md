## 🚨 0단계 — 가장 먼저: Apps Script 재배포 (사장님이 직접, 1회)

**이걸 먼저 하지 않으면 공지 보내기 / 학생 선택이 동작하지 않습니다.**

대상: **지필고사 리포트 구글시트**의 Apps Script

1. 지필고사 리포트 구글시트 열기 → 상단 **확장 프로그램 → Apps Script**
2. 편집기의 기존 코드를 전부 지우고, 저장소의 **`shueguk-report/backend-createReport.gs`** 최신 내용을 붙여넣기 → **저장(Ctrl+S)**
3. 우측 상단 **배포 → 배포 관리** → 기존 배포의 **연필(편집)** 아이콘 → 버전 **"새 버전"** 선택 → **배포**
   - ⚠️ 반드시 "배포 관리 → 편집 → 새 버전" 경로로! (새 배포를 만들면 URL이 바뀌어 다른 페이지가 깨집니다.)
4. 배포 후 URL이 그대로인지 확인 (`.../exec`)

확인 방법: 브라우저에서 `…/exec?action=roster` 열었을 때 학생 명단 JSON이 나오면 성공.

## 1단계 — 학생 선택 시스템을 3개 도구에 적용

공용 위젯은 이미 완성됨: `shueguk-report/student-picker.js` + `student-picker.css`
(전 학년 / 학년 / 개인 / 일부 선택. `picker.getSelection()` → `{type, target, count, summary}`)

붙이는 코드(어느 페이지든 동일):
```html
<link rel="stylesheet" href="https://coke4497-sys.github.io/shueguk-report/student-picker.css">
<div id="picker"></div>
<script src="https://coke4497-sys.github.io/shueguk-report/student-picker.js"></script>
<script>
  var picker = StudentPicker.create(document.getElementById('picker'));
  // 제출 시: var sel = picker.getSelection();
</script>
```

### 적용 대상 (각 도구별로 "선택한 학생에게 무엇을 할지" 먼저 정하기)
- [ ] **주말 실전 모의고사** — 선택한 학생에게 무엇을? (예: 응시 배정 / 개별 공지 / 신청 링크 발송 대상 지정?) → 내일 확정
- [ ] **H WORK** — 선택한 학생에게 과제 배정 / 공지? → 내일 확정
- [ ] **클리닉** — 선택한 학생에게 클리닉 배정 / 공지? → 내일 확정

> 위젯(프런트 UI)은 그대로 재사용 가능. 각 도구는 **선택 결과(type/target)를 받아서 무엇을 할지**(어느 시트/탭에 어떻게 저장할지)를 백엔드에 추가해야 함. 도구마다 백엔드(Apps Script)가 다를 수 있으니 각각 확인 필요.

## 2단계 — (다음 작업) 학생용 페이지 "읽지 않음" 표시

학생용 페이지 메뉴에서, **확인하지 않은 내용이 있으면 읽지 않은 메시지(빨간 점/뱃지) 표시**.
**확인 후에는 사라짐.**

검토할 점(내일 설계):
- "읽음" 상태를 어디에 저장? (학생 기기 localStorage 기준 / 서버 기록 기준)
  - localStorage: 간단하지만 기기 바꾸면 다시 안 읽음 표시
  - 서버: 정확하지만 백엔드에 읽음기록 저장 필요
- 어떤 항목에 적용? (알려드립니다 공지 / 성적 리포트 / 클리닉 등)
- 뱃지 위치·디자인
