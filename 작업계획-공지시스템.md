## ✅ 완료된 것

**공용 학생 선택 위젯** — `shueguk-report/student-picker.js` + `.css`
- 모드: 전 학년 / 학년(여러 학년) / 개인 / 일부
- 일부 탭에 **학년·학교 필터 + "보이는 학생 모두 선택"** → 특정 학년+특정 학교만 한 번에 선택
- 사용법:
  ```html
  <link rel="stylesheet" href="https://coke4497-sys.github.io/shueguk-report/student-picker.css">
  <div id="picker"></div>
  <script src="https://coke4497-sys.github.io/shueguk-report/student-picker.js"></script>
  <script>
    var currentSel = null;
    var picker = StudentPicker.create(document.getElementById('picker'), {
      endpoint: REPORT_API,            // 리포트 백엔드 .../exec
      onChange: function(sel){ currentSel = sel; }
    });
    // 제출 시: var sel = currentSel || picker.getSelection();  // {type, target, count, summary}
  </script>
  ```
  ⚠️ **반드시 onChange의 currentSel을 쓸 것** (picker.getSelection() 직접 호출이 일부 환경에서 null 반환하던 이슈 회피 — 검증됨)

**중앙 배정 저장소** (리포트 시트, 모든 도구 공용)
- `배정` 탭: A작성일 B도구 C항목 D대상유형 E대상 F마감 G비고
- 백엔드 액션: `addAssignment` / `assignList?tool=&item=` / `deleteAssignment` (pw=`sh` 필요)
- 항목(item) 규칙: H WORK = `"강사 / 제목"`

**공지(알려드립니다)** — `notice.html` + 백엔드 공지 CRUD + 슈국 티쳐스 "공지 보내기" 카드. 입력+노출/숨김+삭제.

**H WORK 배정·관리 (표준 템플릿, 검증 완료)**
- `shueguk-h-work/hwork_assign.html`: 과제 선택 → 학생 배정 → 제출/미제출 현황(이름+학교 매칭, 동명+동학교는 ⚠️ ID확인)
- 허브 `shueguk-hub/hwork.html`에 "배정·현황" 카드
- 학생 개인 페이지 `shueguk-report/s.html`: 'H-work' 카드가 **그 학생에게 배정된 과제만** 표시 (백엔드 `collectAssignments_` + resp.homework)

## ✅ 추가 완료 — 지필고사 리포트(분석지) 시스템
- **허브 3메뉴**: `shueguk-hub/report.html` — ①제작(m.html) ②수정·배정(analyses.html) ③학생작성 확인(t.html)
- **제작+배정 통합**: `m.html` — 등록 시 student-picker로 학생 배정(전체/학년/개인/일부),
  저장 버튼 하나가 리포트+배정을 함께 저장. `?id=` 로 열면 수정 모드(제목=ID 잠금, 원래 ID 유지 → 중복 방지)
- **수정·배정 관리**: `analyses.html` — 리포트 목록+현재 배정 표시, 수정 버튼(→m.html?id=), 배정 해제, **리포트 삭제**
  (백엔드 `deleteReport`: 보고서목록·문항·배정 제거, 제출결과=학생 복기는 보존)
- **배정 저장 방식**: `배정` 탭에 도구=`지필고사 분석지`, 항목=시험ID. 학교·학년 자동배정은 선택과목 문제로 **폐기** — 교사 배정만 유효
- **학생 페이지**: 분석지 복기는 별도 카드가 아니라 **'알려드립니다'에 통합** (미입력=빨간 뱃지+복기 입력 버튼 맨 위, 완료=아래)
- **복기 입력**: `r.html` — 상단 시험 제목 표시, 리포트 이미지(빈 분석지 PNG) 저장, 제출 응답 검증(실패 시 오류 표시)
