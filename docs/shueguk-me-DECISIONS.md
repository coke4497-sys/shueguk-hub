## D-1. 데이터는 하나로, 코드 저장소는 그대로 (2026-06-14)

**결정**
- 🗄️ **데이터·학생 신원은 통합한다.** 흩어진 도구 결과를 **하나의 중앙 시트(또는 Apps Script)** 에 `학생코드` 기준으로 모은다.
- 📁 **코드 저장소는 합치지 않는다.** 어휘·문법·H WORK·클리닉·OMR은 각자 저장소에 그대로 두고, 학생 앱은 별도 저장소(`shueguk-me`)로 둔다.

**이유**
- 포인트/계급(3단계)을 막는 건 코드 분산이 아니라 **데이터 분산 + 식별 키 불일치**다. 해결해야 할 건 데이터 쪽.
- 기존 도구들은 이미 GitHub Pages로 **live 운영 중**. 저장소를 합치면 URL이 바뀌어 **학생에게 보낸 링크·북마크·설치된 PWA가 전부 깨진다.** 이득(정리된 느낌)보다 비용(중단·마이그레이션)이 크다.

**목표 구조**
```
기존 도구 저장소들  →  그대로 (각자 운영)
        │  (각 도구가 결과를 ↓ 로 흘려보냄: 학생코드 + 활동 + 점수)
        ▼
🗄️ 중앙 데이터 (시트/Apps Script)  ← 학생코드로 통일  ★새로 만드는 핵심
        ▲ (읽기)
        │
🆕 shueguk-me (학생 앱, 자기 저장소)  ←  중앙 데이터 + Notion 읽기
```

**함의**
- 명단(로스터)의 **`학생코드`가 전 시스템을 꿰는 단일 이름표**가 된다. → 미리 채워둘수록 3단계가 쉬워짐.
- "코드도 한 저장소로 정리"는 필요 시 **장기 리팩토링**으로만 고려. 현재 목표엔 불필요.
